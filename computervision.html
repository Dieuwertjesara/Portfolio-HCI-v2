Assignment cv1

With this excersize I have to change the basic example code. In the example code they use face detection and put a green rectangle around a detected face. My assignment is to change it to a circle. I changed the rect code to ellipse. This turned the rectangle in a circle but it wasn't nicely centered around the face annymore. That's why I added +50 to both the x-axis and y-axis. These were calculated by the face location data, but the circle stayed in the upper left corner of the face. Bij adding 50 to the face location data the circle went down to the centre of the face but on the left side still. So I had to add it to the x-axis aswell to get it more centered.

I looked around on the internet why this happened and later realized that by changing the rectacle to an ellipse was not enough. I also had to add the ellipseMode(CORNER) to make it work. I added that and removed the +50's and now it works the way it should.

Assignment cv2

With this assignment I had to change the code so that there would be a black box over the eyes of the person on screen and a red circle on the nose. This took a while and it is not perfect, but it works. I used the face option again instead of the eyes for the black bar, because this way it would be one sollid bar instead of two rectangles. This is because when you use the eye function it applies it per eye instead of per pair of eyes. I did use CASCADE_EYE aswell, because this way the bar starts at the height of the registered eyes. I first tried to just use a face covering rechtangle and scale it down to start at 1/3 or 1/4 of the face, but this did not always cover the eyes from every distance or on every face since the eyes don't always start on 1/3 or 1/4 of everyone's face. I did use /3 for the height of the bar because this rougly translated to for sure go over the bottom part of the eye so the eye wouldn't be visible, but the bar also wouldn't be covering the nose or mouth. It basically shows 1 third off the length of the face so this should be applicably to most faces. The nose is in the same for loop as the rest because this for some reason made the nose more stable. Before this I'd put it in a seperate for loop, but it detected multpiple things as noses so it was very glitchy. Now the nose mostly stays put and there's minimal extra noses, but it does cause an error if I move to much to the sides or if I move to far to the back. I'm still happy with the result because even if it works with limits, I still got it to work better than before.
